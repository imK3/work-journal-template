# 个人工作记录管理系统（模版）

一套基于 Claude 的个人工作管理系统。你只需要往 INPUT.md 里写工作记录，Claude 自动帮你分类归档、生成日报周报、维护待办计划。

## 核心理念

**你负责记录，Claude 负责整理。**

写的时候不用管格式、不用想分类，随手写就行。Claude 会：
1. 解析你写的内容，判断属于哪个事项和分类
2. 为每个事项维护一份五维摘要（为什么做、做了什么、结果如何、时间线、后续待办）
3. 自动同步待办到 PLAN.md，过期自动顺延
4. 生成日报和周报
5. 归档已处理的输入

## 快速开始

1. **初始化**：按 `SETUP.md` 中的引导，用 Claude 一键生成你的定制版
2. **日常使用**：编辑 `INPUT.md`，写下你做的事，用 `---` 分隔每条
3. **查看计划**：打开 `PLAN.md`，看未来几天要做什么
4. **定时整理**：用 Claude Cowork 的 Schedule 功能定时执行 `PROMPT.md`，或手动触发

## 目录结构

```
{系统名称}/
├── INPUT.md              ← 你写工作记录的地方（唯一入口）
├── PLAN.md               ← 未来 N 天计划 + 各事项待办汇总
├── INPUT_归档/            ← 已处理的输入按周归档
├── README.md             ← 你正在看的这个文件
├── PROMPT.md             ← 定时整理的 Prompt（配置定时任务用）
├── SETUP.md              ← 初始化引导（用完可删）
├── 日报/                 ← 按月/按天的日报
│   └── YYYY-MM/
│       └── YYYY-MM-DD.md
├── 周报/                 ← 按 ISO 周的周报
│   └── YYYY-WXX.md
├── 事项/                 ← 按分类管理的具体事项
│   ├── {分类1}/
│   │   └── {事项名称}/
│   │       ├── README.md    ← 事项五维摘要
│   │       └── 附件/        ← 相关文件
│   ├── {分类2}/
│   ├── ...
│   └── _归档/            ← 已完成事项存档
└── .系统/                ← 系统文件（自动维护）
    ├── 配置.md           ← 你的自定义配置
    ├── 事项模板.md        ← 事项摘要的模板
    └── 上次检查快照.md    ← 整理状态记录
```

## 事项的五维摘要

每个事项自动生成包含以下维度的 README.md：

| 维度 | 说明 |
|------|------|
| 为什么做 | 背景和动机 |
| 做了什么 | 具体动作和关键决策 |
| 结果如何 | 产出、成效、遇到的问题 |
| 进度时间线 | 每次更新的时间轴记录 |
| 后续待办与建议 | 接下来做什么 + 改进方向（自动同步到 PLAN.md） |

## 计划管理（PLAN.md）

- 打开 PLAN.md 即可看到未来几天要做的事
- 各事项的后续待办汇总在一起，方便全局掌控
- 过期未完成的待办自动顺延到当天
- 待办完成后打 `[x]` 会同步记录到事项的进度时间线

## 附件管理

- 每个事项是一个文件夹，包含 `README.md`（摘要）和 `附件/` 子目录
- 在 INPUT.md 中贴文件路径，Claude 会自动复制到对应事项的附件目录
- 附件清单在事项的 README.md 中维护

## INPUT.md 写法示例

```markdown
---

今天跟客户开了需求评审会，确认了 v2.0 的核心功能：用户画像和推荐系统。
客户希望月底前看到原型。会议纪要在 ~/Downloads/需求评审会0415.pdf

---

修了一个线上 bug，用户注册时邮箱验证码发不出去。
原因是 SMTP 配置里端口写错了（465 → 587）。已上线。

---

帮新来的实习生 review 了代码，主要问题是错误处理太粗糙，
全用 catch(e) {} 吞掉了。给他讲了一下 error boundary 的概念。
```

不用写日期、不用选分类，Claude 会自动处理。

## 定制化

需要调整系统配置时（改分类、改计划天数、改日报格式等），直接告诉 Claude 即可，它会帮你更新。

首次使用建议配合 `SETUP.md` 完成初始化，Claude 会根据你的角色和工作类型自动生成完整配置。
