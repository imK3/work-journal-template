# 初始化引导

> 这个文件帮你用 Claude 一键生成定制化的工作管理系统。
> 你有两种方式：**交互式**（Claude 问你答）或**一句话式**（直接告诉 Claude 全部信息）。

---

## 方式一：交互式初始化

把以下 Prompt 发给 Claude：

```
请帮我初始化工作管理系统。我的模版在 {这个文件夹的路径}。
请依次问我以下问题，然后根据我的回答完成配置：

1. 你的工作角色是什么？（如：后端工程师、产品经理、安全分析师）
2. 你的工作主要分成哪几类？（建议 3~6 个，如：产品设计、技术开发、团队管理）
3. 你希望系统叫什么名字？（会用于文件夹名和标题）
4. 你希望把工作文件夹放在哪里？（默认 ~/Documents/）
5. 你希望多久整理一次？（默认每天早上 9:00）
6. PLAN.md 展示未来几天的计划？（默认 3 天）

收集完后，请执行以下操作：
- 将整个模版文件夹复制到我指定的位置，用系统名称命名
- 根据我的回答填充 .系统/配置.md
- 在 事项/ 下为每个分类创建子文件夹
- 更新 PROMPT.md 中的工作目录路径
- 更新 README.md 中的目录结构和分类说明
- 初始化 git 仓库
```

## 方式二：一句话初始化

把模版路径和所有信息一次性告诉 Claude：

```
请基于 {模版路径} 帮我创建工作管理系统：
- 角色：{你的角色}
- 系统名称：{名称}
- 存放位置：~/Documents/{名称}
- 分类：{分类1}、{分类2}、{分类3}、...
- 整理频率：每天早上 9:00
- 计划天数：3

请复制模版到目标位置，填充配置，创建分类文件夹，初始化 git。
```

### 示例

```
请基于 ~/Downloads/work-journal-template 帮我创建工作管理系统：
- 角色：全栈工程师
- 系统名称：2026产品工作
- 存放位置：~/Documents/2026产品工作
- 分类：产品设计、前端开发、后端开发、运维部署、团队协作
- 整理频率：每天早上 9:00
- 计划天数：3

请复制模版到目标位置，填充配置，创建分类文件夹，初始化 git。
```

## 方式三：复制后手动改

如果你不想用 Claude 初始化，也可以：
1. 把这个文件夹复制到你想要的位置
2. 手动编辑 `.系统/配置.md`，填入你的分类和偏好
3. 在 `事项/` 下手动创建分类子文件夹
4. 编辑 `PROMPT.md` 顶部的工作目录路径

---

## 配置定时任务

初始化完成后，你需要配置定时执行。以下是几种方式：

### Claude Cowork Schedule（推荐）
在 Claude Desktop 中使用 Cowork 的 Schedule 功能，设定定时触发，将 PROMPT.md 的内容作为执行指令。

### 手动执行
随时在 Claude 中打开工作目录，把 PROMPT.md 的内容发给 Claude 即可。

---

## 初始化完成后

- 删除本文件（SETUP.md），它只在初始化时有用
- 开始在 INPUT.md 里写你的工作记录
- 等 Claude 执行第一次整理，检查输出是否符合预期
- 需要调整配置时（改分类、改计划天数等），直接告诉 Claude，它会帮你更新
